### Assignment 5
Refactoring is the process of restructuring an existing body of code to improve its internal structure and quality without altering its external behavior.

This week we are going to refactor the code you have written! You can pick your assignment from week2 or week4 for this. 

Requirements:
- Use main function
- Define some constants/variables at the top of your program.
- Define at least one function that takes arguments and returns a value
- Save the new file as a `.py` file. You can name it any way you like but no space in the file name! Put the refactored file inside a week5 folder, which shall be inside your assignments folder!!!

Think about the following:
- What can be modularized as a function?
- What makes sense to be defined as local/global variables?
- Which data type fits your variable better? 
- How can you structure your code so that it is easier/faster for someone to grasp what is going on? (for example, break down your code into multiple scenes)
- Is it easier to test and debug your code now?

Pay attention to:
- Naming conventions
- Try to avoid very long code inside a function
- Write comments to explain your code
- Beautify your code inside PyCharm : In the main menu, go to **Code | Reformat Code**

* If your original code is rather simple, or if you already used functions, try to expand it a bit more to practice refactoring and fulfill the above requirements with it.
